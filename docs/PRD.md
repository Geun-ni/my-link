# 제품 요구사항 정의서 (PRD) - 마이링크 (MyLink)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 사용자가 자신의 여러 SNS, 웹사이트 등의 링크를 하나의 통합된 프로필 페이지로 모아서 공유하는 가장 심플한 서비스.
- **대상 사용자**: 링크 통합 관리가 필요한 크리에이터 및 일반 사용자.

## 2. 핵심 방향성 (컨셉)
- **디자인/UI**: 테마 선택 기능 없이 `shadcn/ui`의 단일하고 깔끔한 모던 UI 하나로 승부.
- **간편한 수정(인라인)**: 텍스트(이름, 소개글, 링크)를 누르면 그 자리에서 바로 수정할 수 있는 **인라인 편집(Inline Edit)** 시스템.
- **데이터베이스 모델링**: Firebase Cloud Firestore.
  - `users` (최상위 컬렉션)
  - `links` (각 유저의 개별 링크 데이터를 담는 서브 컬렉션)

## 3. 핵심 기능 (MVP)

1. **사용자 인증 및 URL 매핑 (Zero-Onboarding)**
   - 구글 소셜 로그인 (Firebase Authentication).
   - **자동 `displayName` 추출**: 사용자를 귀찮게 하는 별도 가입 폼 없이, 구글 로그인 시 반환된 지메일(Email) 주소의 `@ 앞부분`을 추출하여 초기 **`displayName`**으로 저장합니다.
   - **URL Slug 매핑**: 이 `displayName`이 곧 사용자의 고유 공유 프로필 주소가 됩니다 (예: `도메인.com/지메일아이디`).
2. **프로필 설정 (인라인 편집)**
   - 대시보드 화면에 노출된 `displayName`과 소개(`bio`)를 마우스로 클릭해 즉각 수정.
   - `displayName` 수정 시 사용자의 주소(Slug) 또한 변경되므로 저장 전 중복 검사를 진행.
   - 프로필 사진은 구글 제공 이미지를 차용 (이미지 업로드 X).
3. **링크 관리 및 구글 파비콘 API (인라인 편집)**
   - 인라인 텍스트 입력을 통한 간편한 링크 추가/수정/삭제.
   - **Google Favicon API 연동**: 추가된 링크의 URL 도메인을 바탕으로 구글의 공식 파비콘 API(`s2.googleusercontent.com/s2/favicons?domain=...`)를 불러와 링크 버튼 좌측에 자동 아이콘 생성.
4. **추후 추가 예정 항목**
   - 링크 클릭을 집계하는 조회수 통계 기능 (이후를 대비해 모델링 여백만 마련).

## 4. 프론트엔드 뷰 설계
- **대시보드 (통합 관리 뷰)**: 복잡한 레이아웃 제외, 글씨를 마우스로 직접 콕콕 눌러 편집하는 노션(Notion) 스타일 화면.
- **퍼블릭 프로필 (접속 뷰)**: `도메인.com/displayName`으로 접근 시 화면 상단에 해당 이름이 노출되며, 모바일 친화적인 극도의 심플한 링크 리스트를 제공.
