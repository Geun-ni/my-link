# 사용자 시나리오 (User Scenario) - 마이링크

## 1. 방문자 과정 (Visitor Scenarios)

- 사용자는 **소유자의 다양한 웹페이지(SNS, 포트폴리오 등)를 한곳에서 확인**하기 위해 **제공된 마이링크 URL(`도메인.com/닉네임`)을 클릭하여 프로필 페이지에 접속**한다.
- 사용자는 **해당 페이지가 누구의 것인지 식별**하기 위해 **페이지 상단의 닉네임(`displayName`)과 구글 프로필 이미지, 한 줄 소개(Bio)를 읽어본다.**
- 사용자는 **시각적으로 원하는 사이트를 빠르게 탐색**하기 위해 **목록형으로 나열된 각 링크 버튼 좌측의 파비콘(Favicon) 로고를 살펴본다.**
- 사용자는 **관심 있는 특정 채널이나 웹페이지로 직접 진입**하기 위해 **해당 링크 버튼을 클릭**한다. 
  - *(백그라운드 액션: 클릭하는 순간 시스템은 해당 링크의 '클릭 조회수'를 +1 카운트한다.)*
- 사용자는 **이 유용한 프로필 페이지를 타인에게도 전달**하기 위해 **페이지에 위치한 '공유(링크 복사)' 아이콘을 클릭하여 클립보드에 주소를 복사**한다.

---

## 2. 소유자 과정 (Owner Scenarios)

### 2.1 사이트 가입 및 로그인 (Zero-Onboarding)
- 사용자는 **본인의 통합 링크 프로필을 처음 접하거나 다시 관리**하기 위해 **메인 화면에서 '구글 소셜 로그인' 버튼을 클릭**한다.
- *(가입 시 시스템은 이메일 앞부분을 고유 URL로 부여하여 추가 화면 없이 바로 대시보드로 통과시킨다.)*

### 2.2 프로필 정보 관리 (인라인 에디팅)
- 사용자는 **모니터에 타인에게 띄워질 자신의 이름을 본명이나 다른 닉네임으로 바꾸기** 위해 **화면 중앙에 적힌 닉네임 글자를 바로 클릭(인라인 포커스)하여 새로운 이름으로 고쳐 쓴 후 엔터나 여백을 클릭(Blur)해 저장**한다. 
  - *(저장 시, 이 이름이 고유 공유 URL 주소에도 즉각 반영됨을 인지한다.)*
- 사용자는 **방문자에게 본인을 더 잘 어필**하기 위해 **이름 밑의 빈 소개글(Bio) 영역을 클릭하여 자기소개 텍스트를 적어 넣고 인라인으로 즉시 저장**한다.

### 2.3 링크 리스트 관리 (인라인 에디팅)
- 사용자는 **자신의 새로운 포트폴리오 주소를 목록 제일 위에 추가**하기 위해 **'🔗 링크 추가' 버튼을 눌러 빈 텍스트 영역이 담긴 새 항목 라인을 생성**한다.
- 사용자는 **방문자가 해당 링크가 어떤 내용인지 알게** 하기 위해 **생성된 빈 링크 항목의 '제목 추가' 부분을 마우스로 클릭하고 글씨를 입력하여 즉히 저장**한다.
- 사용자는 **실제 버튼을 눌렀을 때 넘어가게 될 목적지를 세팅**하기 위해 **링크 항목의 'URL 추가' 부분을 클릭하여 실제 주소(예: github.com/...)를 기재한 뒤 저장**한다. 
  - *(저장 직후, 시스템이 URL 도메인을 읽어 구글 파비콘 API를 통해 버튼 옆에 아이콘을 달아주는 것을 시각적으로 확인한다.)*

### 2.4 기존 링크 수정 및 삭제
- 사용자는 **과거에 등록했던 블로그 주소가 바뀌어 이를 업데이트**하기 위해 **별도의 수정 창을 열지 않고 화면에 있는 블로그 URL 텍스트를 마우스로 직접 콕 찍어 새로운 주소로 인라인 타이핑 후 확정**한다.
- 사용자는 **더 이상 노출하고 싶지 않은 오래된 이력서 링크를 제거**하기 위해 **해당 링크 줄 맨 끝에 자리 잡고 있는 자그마한 '휴지통(삭제)' 아이콘을 클릭하여 화면에서 날려 보낸다.** (심플함을 지향하여 확인 모달 없이 직관적으로 삭제됨)

---
*(이 모든 시나리오는 직관적인 '인라인 텍스트 클릭' 방식으로 일관성 있게 구성되며, 페이지 뷰 수 등 복잡한 데이터 없이 오직 **'링크 버튼 클릭'**만을 타겟하여 통계(Count) 처리합니다.)*
